# PRODUCT.md — TermsDesk

register: product

## Product purpose

TermsDesk는 회사가 **자사의 약관·정책 문서를 등록·버전 관리·게시·감사**하는 시스템이다.
회사를 대신해 약관을 작성하거나 법률 자문을 하지 않는다. 이미 회사가 가진 문안을
"버전 관리되는 단일 출처(system of record)"로 보관하고, 게시 시점에 본문을 동결(content-hash)해
변조 불가능한 형태로 외부 앱에 제공하며, 누가 어떤 버전에 동의했는지 영수증으로 남긴다.

SaaS(멀티테넌트)와 사내 설치(self-hosted, 싱글테넌트) 두 형태로 제공된다.

## Users

- **법무·컴플라이언스 담당자 (주 사용자)**: 어떤 약관 버전이 현재 발효 중인지, 변경 이력은
  무엇인지, 규제 감사에 대비한 증거가 있는지 확인하고 게시를 승인한다. 벌금 회피가 동기.
- **개발자**: 약관 게시 API/SDK를 앱에 임베드하고, 동의 영수증을 기록한다.
- **운영/PO**: 정책 초안을 작성하고 게시 워크플로를 거친다.

사용 맥락: 업무 시간, 사무실 모니터 + 가끔 모바일 확인. 작업 집중·신뢰가 최우선.

## Tone

신뢰감, 정밀함, 차분함. "등록부/원장(ledger)"의 무게. 과장 없는 프로페셔널.
레퍼런스: Linear, Vercel, Stripe 대시보드의 절제된 밀도. 안티레퍼런스: SaaS-크림 그라데이션,
hero-metric 템플릿, 장식적 모션, 컴플라이언스 클리셰(네이비+골드 / 틸).

## Strategic principles

- **증거가 제품이다.** "누가/어떤 버전(해시)에/언제/어떻게 동의했는가"를 1분 안에 증명·재현.
- **게시본은 불변.** 한번 게시되면 본문·해시는 절대 바뀌지 않는다. UI도 이 불변성을 시각적으로 표현.
- **컴플라이언스를 주장하지 않는다.** "기록·버전·증거 인프라"로 포지셔닝. "compliant/legal" 단어 회피.
- **밀도는 미덕.** 표·타임라인·라벨이 많은 화면. 사용자가 필요로 하는 정보를 숨기지 않는다.
